# Mapa de Componentes

## Componentes Reutilizáveis
### ActionButtons
- propósito: ações rápidas padronizadas.
- páginas: condomínios, técnicos, visitas, relatórios, contratos.

### ConfirmationModal
- propósito: confirmar ações destrutivas.
- páginas: condomínios, técnicos, visitas, relatórios, contratos.

### ContractPreview
- propósito: prévia visual do contrato.
- páginas: contratos, detalhe do condomínio.

### EmptyState
- propósito: estado vazio padronizado.
- páginas: várias listagens e seções.

### FileUploader
- propósito: upload de arquivo único.
- páginas: contratos, detalhe do condomínio.

### FilterPanel
- propósito: painel de filtros.
- páginas: condomínios, técnicos, visitas, relatórios, contratos.

### FormField
- propósito: estrutura consistente de campo e label.
- páginas: todos os formulários.

### ModalShell
- propósito: shell de modal genérico.
- páginas: detalhes e prévias em todo o sistema.

### PageHeader
- propósito: cabeçalho padrão.
- páginas: praticamente todas as páginas internas.

### PhotoUploader
- propósito: upload múltiplo de fotos.
- páginas: formulário de visita.

### ReportPreview
- propósito: visualização consolidada do relatório.
- páginas: visitas, formulário de visita, relatórios.

### SectionCard
- propósito: agrupamento visual por seção.
- páginas: aplicação inteira.

### StatCard
- propósito: indicadores do painel.
- páginas: dashboard.

### StatusBadge
- propósito: padronização visual de status.
- páginas: toda a aplicação.

### ToastStack
- propósito: feedback de sucesso/erro.
- páginas: renderizado no layout autenticado.

## Oportunidades Futuras
- `DataTable` com paginação e ordenação server-side.
- `Tabs` reutilizáveis para detalhes complexos.
- `EntityDrawer` para edição lateral.
- `UploadList` para anexos múltiplos.
- `PrintLayout` unificado para documentos.
